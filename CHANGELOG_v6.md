# Семён v7.1 — роль «Сотрудник» (запись встреч)

Доступ к боту для сотрудника с ограниченными правами: только найти свободный слот и записать встречу.

**Что умеет сотрудник:**
- «Какие слоты свободны в пятницу?» → бот считает свободное время в рабочие часы и показывает.
- «Поставь консультацию с Иваном в 15:00» → бот ставит встречу в календарь.

**Что сотрудник НЕ видит и не может:**
- Задачи, финансы (KPI), планы недели/месяца, узкое место, идеи, проекты — полностью скрыты
  (отдельный промпт + урезанный набор инструментов: только find_free_slots и add_calendar_event).
- Названия чужих встреч не раскрываются — показывается только свободное время.
- Команды владельца (/post, /dedup, /evening, /clear и т.д.) для сотрудника заблокированы.

**Уведомление владельцу:** когда сотрудник ставит встречу, тебе в личку прилетает
«📅 Сотрудник записал встречу: … — …».

**Настройка (Railway → alta-assistant → Variables):**
```
ASSISTANT_USER_IDS = 464253471        # Telegram id сотрудника (через запятую если несколько)
WORK_START   = 10:00                  # необяз. (по умолч. 10:00)
WORK_END     = 19:00                  # необяз. (по умолч. 19:00)
SLOT_MINUTES = 60                     # необяз. (по умолч. 60)
```
Сотрудник пишет боту `/start` — получит короткую инструкцию для записи.

# Семён v7.0 — Google как источник правды + живые планы

Два бага из боевого теста («ты ничего не сделал» при выполненных задачах и ссылки на прошлую неделю)
были архитектурными. Исправлено в корне.

**Задачи — теперь ТОЛЬКО Google Tasks (источник правды).**
- Бот больше не ведёт собственный список задач (локальная база-дублёр отключена).
- Статус «сделано/не сделано» берётся из Google Tasks. Отметил в Google → бот это видит.
- Системный промпт, утренний план, вечерняя сводка читают задачи и выполнение из Google
  (срез кэшируется 60 сек; после добавления/закрытия задача видна сразу).
- Убраны мёртвые локальные тул-команды (list_tasks/complete_task/delete_task/update_task).
- Это убирает «3-дневный мусор», который тянулся из старой базы.

**Планы недели и месяца — живые, «последний победил».**
- Отдельное хранилище `plans` (week/month). Новый план перезаписывает старый.
- Бот читает план ТОЛЬКО отсюда, не из истории переписки → больше не ссылается на прошлую неделю.
- Прислал новый план (в т.ч. в ветке ПЛАН НЕДЕЛИ) → бот сохраняет через set_plan.

**Прочее в этом круге правок:**
- Каскад моделей с фолбэком при лимите 429 (v6.4) и устойчивый парсинг списка моделей (v6.5).
- «Железное правило №1»: просили добавить — вызывай инструмент сразу, без «сейчас сделаю» (v6.3).
- Чистка вывода от тех-мусора, изоляция контекста по веткам, дедуп (v6.1).

**Что осталось на твоей стороне (один раз):**
- OAuth consent screen → **In production**, иначе refresh-токен Google умрёт через 7 дней.
- По желанию — billing в Google AI для стабильной работы сильной модели.

# Семён v6.2 — починка записи в Google Calendar

**Проблема:** Семён не мог сохранять события («не получилось сохранить в календарь»).
В логах: `credentials do not contain the necessary fields need to refresh the access token`.
Сохранённый `token.pickle` не содержал `client_id/client_secret/token_uri`, поэтому
по истечении access-токена (~1 час) его не получалось обновить — чтение ещё работало, запись падала.

**Фикс (код):**
- Общий загрузчик токена `_load_google_creds` достраивает недостающие поля для refresh
  из переменных окружения `GOOGLE_CLIENT_ID` / `GOOGLE_CLIENT_SECRET`, после чего refresh
  снова работает, а полные creds сохраняются обратно в `token.pickle`.
- Толерантный разбор даты события: принимает `2026-06-04 15:00`, `04.06.2026 15:00`,
  `4.06 15:00`, ISO — модель больше не спотыкается на формате.
- Ошибка создания события теперь возвращается наверх (видна причина), а не «глотается».

**Что нужно сделать (Railway → alta-assistant → Variables):**
```
GOOGLE_CLIENT_ID     = <твой OAuth Client ID из Google Cloud Console>
GOOGLE_CLIENT_SECRET = <твой OAuth Client Secret>
```
Берутся в Google Cloud Console → APIs & Services → Credentials → твой OAuth 2.0 Client ID
(тот же, которым генерировался токен).

**Если после этого календарь всё ещё не пишет** — значит в `token.pickle` нет и `refresh_token`.
Тогда нужно перегенерировать токен с offline-доступом (`access_type=offline`, `prompt=consent`)
и обновить `GOOGLE_TOKEN_BASE64`. Напиши — помогу со скриптом авторизации.

# Семён v6.1 — фиксы качества (поверх v6.0)

Три проблемы из боевого теста закрыты:

1. **Технический мусор в сообщениях.** Постобработка `clean_ai_output` вырезает напечатанные
   вызовы инструментов (`add_idea(...)`), служебные пометки «(зафиксировал в банке)»,
   markdown-заголовки `###`, линии `---` и кривые маркеры — заменяет `*`/`-` на `•`.
   Плюс жёсткий запрет в промпте печатать вызовы инструментов и заголовки.
2. **Контекст не разделялся по веткам.** История теперь изолирована по conv-ключу
   (личка и каждая тема-ветка группы — отдельный контекст). Старая общая история
   автоматически переносится в личку. Вопрос в «План дня» больше не тянет ответы из других веток.
3. **Дубляж задач.** Нормализованная дедупликация: «написать КП», «отправить КП Павлу»,
   «КП Павлу» = одна задача. Дубли не создаются, а существующие чистятся командой `/dedup`
   или инструментом `dedup_tasks` (локально + Google Tasks).

**После деплоя** один раз выполни `/dedup` — почистит накопившиеся дубли.

# Семён v6.0 — апгрейд до AI-операционного директора

Семён перестал быть «умным планировщиком». Теперь это ассистент-операционник,
который держит в голове цели, деньги и узкое место, спорит с владельцем и каждый
вечер думает как стратег — как выйти на новый уровень.

## Что нового

**Новые сущности (БД, миграция автоматическая и безопасная)**
- Цели: месяц / неделя / фокус дня / узкое место (`goals`).
- Проекты с рейтингом рычага = прибыль × вероятность × стратегичность ÷ время (`projects`).
- Банк идей — второй мозг: контент / продукт / партнёрство / автоматизация (`ideas`).
- Ожидания от других людей (`waiting_for`).
- У задач появился тип нагрузки `energy_type` (deep/comm/ops/creative/routine) и привязка к проекту.

**Финансовый контур**
- KPI из task-board (раздел «Экономика») приходят в системный промпт и инструмент `get_business_kpi`.

**Новая личность**
- Тон живого человека, без канцелярита и шаблонов нейросетей (список запрещённых фраз).
- Роль операционного директора: возвращает Глеба к тому, что двигает бизнес.
- Функция «спорить»: перегруз, потеря фокуса, несоответствие цели, риск, упущенная возможность.
- Разбор сущностей: не всё — задача (идея ≠ задача, ожидание ≠ задача).

**Вечерний стратег (фоном, каждый день)**
- Итог дня (сделано / не успели / переносим / решения / ожидания).
- Взгляд опытного предпринимателя: 2-3 мощные амбициозные идеи на завтра к большой цели.
- Лучшие идеи сохраняются в банк идей.
- Воскресенье — недельный обзор накопленных идей с выбором топ-3.

**Новые команды**
`/focus` `/goals` `/bottleneck` `/projects` `/ideas` `/waiting` `/strategy` `/evening`

## Новые переменные окружения (Railway → alta-assistant)

Все опциональны — есть значения по умолчанию.

```
AUTO_EVENING_ENABLED = true          # вечерняя стратегическая сводка
AUTO_EVENING_TIME    = 21:00         # время вечерней сводки (МСК)
AUTO_IDEAREVIEW_TIME = 19:00         # время воскресного обзора идей
BIG_GOAL             = 10 млн ₽ чистой прибыли   # ориентир для стратега
```

Плюс уже добавленные для финансового моста:
```
FINANCE_API_URL   = https://alta-production-82f7.up.railway.app/api/kpi
FINANCE_API_TOKEN = <SEMYON_TOKEN из task-board>
```

## Миграция
- При старте `init_db()` создаёт новые таблицы и добавляет колонки в `tasks` идемпотентно.
- Существующие задачи и история полностью сохраняются. Откат не требуется.
- Зависимости не менялись (всё на стандартной библиотеке).
